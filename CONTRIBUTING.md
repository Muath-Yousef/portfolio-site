# Contributing

Thank you for contributing.

## Before you start
- Search existing issues and pull requests before opening new work.
- Keep changes focused and easy to review.

## Branch and commit guidance
- Use descriptive branch names.
- Write clear commit messages explaining intent.

## Pull requests
1. Explain what changed and why.
2. Link related issues.
3. Include validation steps and results.
4. Keep scope minimal; split large work into smaller PRs.

## Documentation and security
- Update docs when behavior or usage changes.
- Do not include secrets, tokens, or sensitive data in commits.
- For vulnerabilities, follow `SECURITY.md`.

## Review expectations
PRs are reviewed for correctness, clarity, maintainability, and security impact.
