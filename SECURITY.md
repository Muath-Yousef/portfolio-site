# Security Policy

## Reporting a vulnerability
Please do **not** open public issues for suspected vulnerabilities.

Use GitHub Security Advisories for private reporting:
- Open the repository
- Go to **Security** → **Advisories** → **Report a vulnerability**

If private reporting is unavailable, open a regular issue with minimal detail and request secure follow-up.

## Response expectations
- Initial triage target: within 5 business days
- Status updates provided as investigation progresses
- Valid findings will be remediated based on impact and feasibility

## Scope
This policy applies to source code, workflows, documentation tooling, and repository configuration in this project.
