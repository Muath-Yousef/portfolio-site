# Usage Model

## How this repository is expected to be used

1. Identify a recurring operational task that causes friction.
2. Implement a small, focused utility in `tools/<tool-name>/`.
3. Reuse `shared/` components only when reuse is clear and justified.
4. Document usage, assumptions, and limits in the tool README.
5. Keep implementation simple enough for fast review and safe maintenance.

## Operational orientation

Tools should support day-to-day work for operators, analysts, and engineers who need practical outcomes with minimal setup.
