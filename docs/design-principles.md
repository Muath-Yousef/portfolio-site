# Design Principles

## Core principles

1. **Utility first**: Build tools that solve real work problems.
2. **Lightweight by default**: Avoid unnecessary frameworks and complexity.
3. **Clear boundaries**: Keep each tool narrow in purpose and behavior.
4. **Maintainability**: Optimize for readability, reviewability, and long-term upkeep.
5. **Disciplined reuse**: Use `shared/` for true cross-tool value, not convenience coupling.
6. **Honest maturity**: Do not present placeholders as mature production systems.

## Practical standard

Every addition should improve real workflows while keeping the repository understandable and clean.
