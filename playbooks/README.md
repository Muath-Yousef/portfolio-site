# Playbooks

This directory is for operational playbooks used to support consistent response and investigation execution.

## Scope

Playbooks in this directory should describe:

- scenario context and prerequisites
- triage and validation steps
- containment and remediation actions
- escalation and communication requirements
- closure criteria and post-incident actions

## Usage Model

Playbooks should support both:

- analyst-led execution in manual or semi-automated workflows
- automation-assisted execution where steps can be delegated to tooling

## Quality Standards

- Write steps in clear, unambiguous language.
- Define decision criteria and evidence requirements.
- Keep dependencies explicit (tools, permissions, data sources).
- Update playbooks when workflow or control changes are introduced.
